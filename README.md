# BarBridge Democracy

אתר אינטרנט מחקרי שמרכז:

- לשכות עורכי דין במדינות עם נסיגה דמוקרטית
- שמות ראשי הלשכה ומקורות אימות
- פרטי קשר וקישורים לאתרים רשמיים
- נוסח פנייה רשמי באנגלית לשיתוף פעולה עם לשכת עורכי הדין בישראל
- פירוט צעדי שחיקה דמוקרטית בכל מדינה
- לשונית נפרדת של ארגוני עורכי דין, שופטים ותובעים שפעלו נגד פגיעה בשלטון החוק

האתר נבנה כאתר סטטי קל משקל. אפשר לפתוח אותו ישירות כקובץ מקומי, להריץ אותו עם שרת Python פשוט, או לפרוס אותו כ-Static Site ב-Render.

## הרצה מקומית

להריץ מתוך התיקייה:

```bash
python3 server.py
```

ואז לפתוח את `http://127.0.0.1:8080`.

## פריסה כאתר אינטרנט

הפרויקט מוכן לפריסה כאתר ציבורי.

### Render

1. מעלים את הפרויקט ל־GitHub.
2. יוצרים ב־Render שירות חדש מסוג `Web Service`.
3. מחברים את מאגר ה־GitHub.
4. יוצרים ב-Render שירות חדש מסוג `Static Site`.
5. בוחרים את ה-repository.
6. מגדירים:

```text
Build Command: (leave empty)
Publish Directory: .
```

אם אתה פורס דרך Blueprint, Render יכול לקרוא את [render.yaml](/Users/veredzaitman/Documents/New%20project/render.yaml#L1), שבו האתר מוגדר עכשיו כ-`runtime: static`.

בסיום הפריסה יתקבל קישור ציבורי בסגנון:

```text
https://barbridge-democracy.onrender.com
```

אם Render מציג `Exited with status 127 while running your code`, זה מתאים למצב שבו השירות נוצר כ-`Web Service` ומנסה להריץ פקודת start, למרות שהפרויקט עצמו הוא אתר סטטי. לכן הפתרון הנכון כאן הוא ליצור `Static Site` חדש, לא להמשיך לנסות להרים את אותו Web Service.

## מבנה הפרויקט

- `index.html` : מבנה האתר, לשוניות ותוכן הדף
- `styles.css` : העיצוב, הרספונסיביות והלשוניות
- `app.js` : הנתונים, הרינדור והאינטראקציות בדפדפן
- `server.py` : שרת סטטי מינימלי
- `render.yaml` : הגדרות פריסה ל־Render
- `site.webmanifest` : הגדרות PWA בסיסיות
- `robots.txt` : הגדרות סריקה בסיסיות

כדי לבצע שינוי:

1. עורכים את הקבצים בתיקיית הפרויקט המקומית.
2. בודקים מקומית שהכול עובד:

```bash
cd "/Users/veredzaitman/Documents/New project"
python3 server.py
```

ואז פותחים את:

```text
http://127.0.0.1:8080
```

3. מעלים את השינויים ל־GitHub:

```bash

git add .
git commit -m "תיאור קצר של השינוי"
git push
```

4. Render יזהה את השינוי ויפרסם אוטומטית גרסה חדשה של האתר.
