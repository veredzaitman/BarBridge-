# חקיקון

אפליקציית web קלה לניטור הצעות חוק בעברית, עם מיקוד בהצעות שעלולות לפגוע במשטר דמוקרטי:

- חיפוש וסינון לפי סטטוס, ועדה, רמת סיכון דמוקרטי וסוג פגיעה
- רשימת מעקב אישית הנשמרת ב־`localStorage`
- ציון סיכון, מוקד פגיעה והסבר קצר לכל הצעה
- קישורים לחומרי רקע של ההצעה מאתר הכנסת כשיש להם התאמה במקור הנתונים
- מדדי מצב ופיד פעילות
- חיבור לנתוני CSV יומיים של כנסת פתוחה, המבוססים על ממשקי הכנסת
- fallback מקומי לנתוני דוגמה אם המקור החי לא זמין

## הרצה מקומית

להריץ מתוך התיקייה:

```bash
python3 server.py
```

ואז לפתוח את `http://127.0.0.1:8080`.

הערה: פתיחה ישירה של [`index.html`](/Users/veredzaitman/Documents/New project/index.html) לא תטען את ה־API.

## פריסה כאתר אינטרנט

הפרויקט מוכן עכשיו לפריסה כאתר ציבורי.

### Render

1. מעלים את הפרויקט ל־GitHub.
2. יוצרים ב־Render שירות חדש מסוג `Web Service`.
3. מחברים את מאגר ה־GitHub.
4. Render יזהה את הקובץ `render.yaml` ויפרוס את האתר עם:

```bash
python3 server.py
```

בסיום הפריסה יתקבל קישור ציבורי בסגנון:

```text
https://hakikon.onrender.com
```

השרת תומך גם ב־`PORT` וב־`HOST` מהסביבה, כך שהוא מתאים לשירותי hosting סטנדרטיים.

## איך מעדכנים את האתר

כדי לבצע שינוי באתר:

1. עורכים את הקבצים בתיקיית הפרויקט המקומית.
2. בודקים מקומית שהכול עובד:

```bash
cd "/Users/veredzaitman/Documents/New project"
python3 server.py
```

ואז פותחים את:

```text
http://127.0.0.1:8080
```

3. מעלים את השינויים ל־GitHub:

```bash
cd "/Users/veredzaitman/Documents/New project"
git add .
git commit -m "תיאור קצר של השינוי"
git push
```

4. Render יזהה את השינוי ויפרסם אוטומטית גרסה חדשה של האתר.

### איפה משנים כל דבר

- `index.html` : טקסטים, מבנה הדף והרכיבים
- `styles.css` : צבעים, ריווח, כפתורים ועיצוב
- `app.js` : פעולות בדפדפן, סינון, טפסים ואינטראקציות
- `server.py` : טעינת נתונים, API פנימי ולוגיקת שרת
